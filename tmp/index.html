Modules

Los módulos son piezas funcionales que podemos usar en donde las necesitemos y podemos crearlos o usar los que ya están en Node, esa es la naturaleza de Node, tener modulos que puedan ser utilizados por quien lo necesite. Para usar estos módulos debemos usar require para usar los módulos en donde lo necesitemos.

Core Modules

Son modulos que vienen incluidos dentro de la instalació de node.js y se pueden usar de inmediato mediante el uso de require. e.g. const fs = require('fs');

Local Modules

Son módulos escritos por nosotros y que son usados de forma local, primero los debemos exportar e.g.module.exports = log y después los usamos en donde queremos usando const myModule = require('./mymodule') e.g

Third Party Modules

Son módulos escritos por terceras personas y que podemos instalar y usar cuando queramos. Los usamos de la misma forma que lo hacemos con los módulos CORE, la única diferencia es que los debemos instalar antes. Los modulos CORE vienen dentro de Node.js y cuando instalamos Node también instalamos NPM. Antes de instalar cualquier paquete primero debemos inicializar NPM a través de npm init, se nos van a hacer algunas preguntas relacionadas con la aplicación en la que estemos trabajando, este proceso lo que va hacer es crear al final un archivo llamado package.json en este archivo quederán gurdados los datos del proyecto y ademáas se iran guardando las lista de los módulos o dependencias que vayamos descargando, de esta forma si compartimos el proyecto en una plataforma como GIT no será necesario subir toda la carpeta node_modules sino que solo teniendo este archivo y mediante el comando npm installse descargarán e instalarán todas las dependencias que ahí se encuentren.

Restarting APP with Nodemon

Nodemon es una aplicación que monitorea los cambios que hacemos en nuestra aplicacion y la vuelve a ejecutar de inmediato así que tenemos que hacerlo nosotros cada vez.

npm install nodemon -g

La instalamos con la flag -g porque de esta manera queda instalada de forma global en nuestra máquina y por lo tanto accesible en nuestra linea de comando para cualquier proyecto en que la queramos usar.

Si por ejemplo queremos mantener nuestra aplicació bajo el ojo de nodemon lo que hacemos es lo siguiente: nodemon app.js y listo, cada vez que hagamos un cambio, nuestra aplicacion se volvera a ejecutar.

Getting input from user

El objeto process es un objeto global y por lo tanto está siempre disponible y puede ser llamado desde cualquier parte sin la necesidad de usar require.

Cuando ejecutamos una aplicacion e.g. node app.js también podemos pasar los argumentos que queramos y utilizarlos cuando queramos. Estos argumntos se añaden de la siguiente forma: node app.js list. Aquí añadimos el argumento list y para leerlos podemos mostrar mostrar en la consola a través de console.log, Dentro de app.js agregamos console.log(process.argv) y cuando corremos node app.js list se nos mostrará el contenido del array.

Con el ejemplo anterior podemos obtener diferentes resultados que van a depender del valor que introduzcamos como argumento, si por ejemplo corremos node app.js read veremos en la consola Fetching note.
